# Orion-Sentinel-DataAICore Configuration Guide

Detailed explanation of all environment variables and configuration options.

## Environment File (.env)

The `.env` file is created by the bootstrap script from `env/.env.example`. All service configuration is done through this file.

To edit:
```bash
sudo nano .env
```

---

## System Settings

### TZ - Timezone

```bash
TZ=Europe/Amsterdam
```

Sets the timezone for all containers. Use a valid [tz database timezone](https://en.wikipedia.org/wiki/List_of_tz_database_time_zones).

**Examples:**
- `Europe/Amsterdam`
- `America/New_York`
- `Asia/Tokyo`
- `UTC`

### HOST_IP - Network Binding

```bash
HOST_IP=127.0.0.1
```

The IP address to bind published ports to.

**Options:**
- `127.0.0.1` - Localhost only (most secure, default) - services only accessible from the host itself
- `192.168.x.x` - Specific LAN IP (recommended for LAN access) - services accessible from your local network
- `0.0.0.0` - All interfaces (least secure, not recommended) - services may be accessible from all networks

**Why this matters:** 
- Setting `127.0.0.1` (default) means services are only accessible from the machine itself - you'll need to SSH in or use a remote desktop to access them
- Setting a specific LAN IP (e.g., `192.168.1.100`) allows access from your local network but not the internet
- Setting `0.0.0.0` binds to all network interfaces, which could accidentally expose services on unintended networks

**Recommendation:**
- For initial setup: Keep `127.0.0.1` and access via SSH tunnel
- For LAN access: Change to your server's LAN IP (e.g., `192.168.1.100`)
- Never use `0.0.0.0` unless you understand the security implications

To change:
```bash
sudo nano .env
# Change HOST_IP=127.0.0.1 to your LAN IP
# Then restart: ./scripts/orionctl restart core
```

### DATA_ROOT - Data Storage Location

```bash
DATA_ROOT=/srv/orion/dataaicore
```

Root directory for all application data. The bootstrap script creates the directory structure:

```
/srv/orion/dataaicore/
├── nextcloud/
│   ├── app/       # Nextcloud application
│   ├── data/      # User files
│   └── postgres/  # Database
├── redis/         # Cache data
├── searxng/       # SearXNG config
├── secrets/       # Generated secrets
└── llm/
    ├── ollama/    # LLM models
    └── openwebui/ # Open WebUI data
```

### PUID / PGID - User/Group IDs (Optional)

```bash
PUID=1000
PGID=1000
```

User and Group IDs for file ownership. Find yours with: `id`

## Nextcloud Configuration

### Admin Credentials

```bash
NEXTCLOUD_ADMIN_USER=admin
NEXTCLOUD_ADMIN_PASSWORD=<generated>
```

Initial admin account credentials. Only used on first startup. Change the password in Nextcloud web UI after setup.

**Security:** The password is auto-generated by the bootstrap script. Never commit this to git.

### Database Settings

```bash
POSTGRES_DB=nextcloud
POSTGRES_USER=nextcloud
POSTGRES_PASSWORD=<generated>
```

PostgreSQL database credentials. Auto-generated by bootstrap.

### Redis Settings

```bash
REDIS_PASSWORD=<generated>
```

Redis password for Nextcloud file locking cache.

### Trusted Domains

```bash
NEXTCLOUD_TRUSTED_DOMAINS=192.168.1.100 localhost
```

Space-separated list of domains/IPs allowed to access Nextcloud.

**When to update:**
- Add your server's hostname
- Add domain name for public access (Phase 2)

**Example for public access:**
```bash
NEXTCLOUD_TRUSTED_DOMAINS=192.168.1.100 localhost cloud.yourdomain.com
```

### PHP Settings

```bash
PHP_MEMORY_LIMIT=512M
PHP_UPLOAD_MAX_SIZE=16G
PHP_MAX_EXECUTION_TIME=3600
```

PHP runtime settings for Nextcloud:
- **PHP_MEMORY_LIMIT:** Maximum memory per request (512M is good for most)
- **PHP_UPLOAD_MAX_SIZE:** Maximum file upload size
- **PHP_MAX_EXECUTION_TIME:** Maximum script runtime (seconds)

## SearXNG Configuration

### Port

```bash
SEARXNG_PORT=8888
```

Port to access SearXNG web interface.

### Secret Key

```bash
SEARXNG_SECRET_KEY=<generated>
```

Secret key for SearXNG (used for secure cookies). Auto-generated by bootstrap.

### Settings File

SearXNG configuration is in `stacks/websearch/searxng/settings.yml`.

To edit:
```bash
sudo nano stacks/websearch/searxng/settings.yml
```

Key settings:

```yaml
# Base URL (set to your LAN address)
server:
  base_url: "http://192.168.1.100:8888/"

# Enable JSON API (for Open WebUI integration)
search:
  formats:
    - html
    - json

# Configure search engines
engines:
  - name: duckduckgo
    engine: duckduckgo
    shortcut: ddg
    disabled: false
```

**Important:** After editing settings.yml, restart SearXNG:
```bash
./scripts/orionctl down websearch
./scripts/orionctl up websearch
```

## LLM Configuration (Ollama + Open WebUI)

### Open WebUI Settings

```bash
WEBUI_SECRET_KEY=<generated>
ENABLE_SIGNUP=true
```

- **WEBUI_SECRET_KEY:** Session encryption key (auto-generated)
- **ENABLE_SIGNUP:** Allow new user registration (`true`/`false`)

**Recommendation:** Set `ENABLE_SIGNUP=false` after creating your admin account.

### Ollama Settings

```bash
OLLAMA_HOST=0.0.0.0
```

Network binding for Ollama. By default, Ollama is internal-only (not published to host).

### Open WebUI to SearXNG Integration

Open WebUI can use SearXNG for web search during conversations. Configure this in the Open WebUI web interface:

1. Log into Open WebUI as admin
2. Navigate to **Settings** → **Web Search**
3. Enable web search
4. Configure the search provider:

   | Setting | Value |
   |---------|-------|
   | Engine | SearXNG |
   | URL | `http://searxng:8080` |
   | Query Parameter | `q` |
   | Result Format | `json` |

5. Save settings

**Note:** The URL uses the internal Docker network hostname (`searxng`), not the external IP.

### Pulling Models

Use the helper script:
```bash
./scripts/pull-model.sh llama3.2:3b
```

Or manually:
```bash
docker exec orion_dataaicore_ollama ollama pull llama3.2:3b
```

**Recommended models for CPU:**
- `llama3.2:3b` - Good general-purpose model
- `qwen2.5:3b` - Good for coding
- `phi3:mini` - Microsoft's small model

## Public Nextcloud Configuration (Phase 2)

⚠️ **Only configure if you want internet-accessible Nextcloud**

### Domain Setting

```bash
ORION_DOMAIN=yourdomain.com
```

Your domain name. Nextcloud will be accessible at `cloud.${ORION_DOMAIN}`.

### Prerequisites

Before enabling public-nextcloud profile:

1. **Domain name** registered and owned by you
2. **DNS A record:** `cloud.yourdomain.com` → Your public IP
3. **Router port forwarding:** 443 (HTTPS) → OptiPlex IP
4. **Trusted domains:** Add `cloud.yourdomain.com` to `NEXTCLOUD_TRUSTED_DOMAINS`

### Enable Public Access

```bash
# 1. Set domain in .env
sudo nano .env
# Set: ORION_DOMAIN=yourdomain.com

# 2. Add to trusted domains
# Add: cloud.yourdomain.com to NEXTCLOUD_TRUSTED_DOMAINS

# 3. Start with public-nextcloud profile
./scripts/orionctl down
./scripts/orionctl up nextcloud public-nextcloud
```

### Caddy Configuration

The reverse proxy config is at `stacks/public-nextcloud/proxy/Caddyfile`.

To edit:
```bash
sudo nano stacks/public-nextcloud/proxy/Caddyfile
```

**Default Caddyfile:**
```caddyfile
cloud.{$ORION_DOMAIN} {
    reverse_proxy nextcloud:80

    header {
        Strict-Transport-Security "max-age=31536000; includeSubDomains"
    }
}
```

**Security notes:**
- Caddy automatically obtains Let's Encrypt SSL certificates
- Only Nextcloud is exposed; other services remain internal
- See [SECURITY.md](SECURITY.md) for hardening guidelines

## Advanced Configuration

### Custom Docker Network

The default network is `dataaicore_internal`. To use a custom network:

```bash
# Create network
docker network create my_custom_network

# Edit compose files to use it
sudo nano compose.yaml
```

### GPU Support for Ollama

If you have an NVIDIA GPU:

1. Install [NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/install-guide.html)

2. Edit LLM compose file:
```bash
sudo nano stacks/llm/compose.yaml
```

3. Uncomment the GPU section:
```yaml
services:
  ollama:
    # ... existing config ...
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: all
              capabilities: [gpu]
```

4. Restart:
```bash
./scripts/orionctl down llm
./scripts/orionctl up llm
```

### External Database

To use an external PostgreSQL database instead of the containerized one, update `.env`:

```bash
POSTGRES_HOST=your-external-db-host
POSTGRES_PORT=5432
POSTGRES_DB=nextcloud
POSTGRES_USER=nextcloud
POSTGRES_PASSWORD=your-password
```

Then disable the internal postgres in compose files.
